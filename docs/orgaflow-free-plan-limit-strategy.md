# orgaflow-free-plan-limit-strategy.md

## Starter Plan Limit Strategy for Orgaflow

This document defines the usage and feature limits for the Orgaflow Starter plan.
The goal is to let small teams experience the core workflow while making the
upgrade path obvious when they need more scale, automation, and branding.

---

# 1. Starter Plan Limits

The Starter plan should use these limits (all **lifetime totals per organization**, not monthly rolling windows, unless product policy changes explicitly):

```
Customers: 50
Invoices: 50
Estimates: 50
Users: 3
Items: 50
```

These numbers are generous enough for freelancers and very small teams, but
they still create clear upgrade pressure as the business grows.

---

# 2. What Starter Should Include

The Starter plan should still deliver the core commercial workflow:

- Dashboard access
- Customers
- Items
- Estimates
- Invoices
- Manual payment registration
- Basic document export

The objective is to let users experience the real product, not a crippled trial.

---

# 3. What Starter Should Not Include

The following capabilities should be blocked on Starter:

- Tasks
- Workflow automations
- Online payments through Stripe
- Platform billing features
- Attachments
- Branded documents

This keeps the value proposition simple:

- Orgaflow Starter is for quoting and invoicing at small scale
- Orgaflow Growth is for document polish and premium document workflows
- Orgaflow Scale is for payments, automation, and operational execution

---

# 4. Document Branding Rule on Starter

Starter plan documents may still show the customer's own logo when that branding
feature is supported by the product flow.

However, documents generated from the Starter plan must keep Orgaflow attribution
in the footer.

Required footer rule:

```
Powered by OrgaFlow
https://orgaflow.dev
```

This keeps the free tier useful while preserving brand visibility for the SaaS.

---

# 5. Upgrade Triggers

The upgrade moments should be natural and directly tied to growth:

- When the organization reaches 50 invoices
- When the organization reaches 50 estimates
- When the organization reaches 50 customers
- When the organization reaches 2 users
- When the organization reaches 50 items
- When the team wants tasks or automations
- When the team wants online payments
- When the team wants attachments or fully branded documents

Examples:

```
You reached the limit of 50 invoices.
Upgrade to continue creating invoices.
```

```
Upgrade to Orgaflow Scale to unlock tasks, automations, and Stripe payments.
```

---

# 6. Positioning

The Starter plan should be positioned as:

```
Low-cost starter plan with a 15-day free trial
```

The commercial narrative is:

- Start with a 15-day trial
- Learn the workflow
- Keep the Starter tier or upgrade when scale, branding, payments, or operations become important

---

# 7. Summary

Recommended Starter plan structure:

```
Customers: 50
Invoices: 50
Estimates: 50
Users: 3
Items: 50
```

Included:

- Core estimate and invoice workflow
- Manual payment registration
- Basic document export

Blocked:

- Tasks
- Automations
- Stripe payments
- Platform billing
- Attachments
- Branded documents

Branding rule:

- Starter plan documents must include Orgaflow attribution in the footer with
  https://orgaflow.dev
